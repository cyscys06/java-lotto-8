## 브랜치 사용 관련
- 서로 다른 범주의 브랜치를 여러 개 사용하여 커밋을 작성할 때,
  잘못된 브랜치에서 커밋하는 경우가 많아져 작업 속도가 저해되는 현상이 발생했습니다.
- 작업 속도가 느려지더라도 브랜치를 올바르게 사용하는 것은 배보다 배꼽이 더 큰 경우이고
  무엇보다 과제 마감 기한을 지키지 못할 것이라고 판단했습니다.
- 따라서 부득이하게 하나의 브랜치에서만 커밋을 작성하기로 결정했습니다. 제 개발 실력이 부족한 점 양해 부탁드립니다.

## 설계 구조
- 기존 MVC 패턴에서 Controller가 맡고 있는 역할(제어 및 비즈니스 로직 수행)이 너무 과하다고 생각하여
비즈니스 로직을 수행하는 별개의 Service 파트를 만들어 책임을 분리했습니다.
- DDD(도메인 주도 디자인) 설계의 핵심 개념 중 하나인 Domain을 사용하여 
최상위 도메인을 '로또'로, 하위 도메인을 각각 '구매 로또', '당첨 번호', '상금'으로 정의한 후 설계했습니다.
